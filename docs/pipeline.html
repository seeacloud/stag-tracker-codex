STag Tracker — Processing Pipeline

Stage 1: Capture
Camera Capture USB cam via DirectShow
1280×720 MJPG
cv2.VideoCapture → BGR frame
BGR → Grayscale cv2.cvtColor(COLOR_BGR2GRAY)
gray 用于光流跟踪
Optical Flow Predict OpticalFlowTracker.update_track()
用前一帧 gray 和当前 gray 预测已有 track 的新位置
Lucas-Kanade 稀疏光流 (120 pts/marker)
ROI 决策 detection_rois()
• 无 track / 每 N 帧 → 全帧搜索
• 有 track → 各 track bbox + padding
• 每 5 帧追加一次全帧 fallback
Stage 2: Multi-Pass Detection (per ROI)
StagDetector.detect(frame, rois) 自适应：若 baseline 已覆盖 expected_ids → 跳过后续 pass
Pass 0 — Baseline CLAHE clip=3.5, grid=8
Sharpen: OFF
Scales: 0.75, 1.0, 1.5
ROI super-res: min 140px
Pass 1 — Aggressive CLAHE clip=4.5, grid=8
Sharpen: ON (amount=1.0)
Scales: 0.6, 1.0, 2.0
ROI super-res: min 100px
每个 Pass 内部流程 ① enhance_for_detection(crop, config) — CLAHE / Sharpen
② 对每个 scale 缩放图像
③ stag.detectMarkers(scaled, libraryHD=17)
④ 角点坐标反缩放 + 偏移回原图坐标
Union + Dedupe 合并所有 pass 的 observations
dedupe_observations(iou > 0.1)
同 marker_id + 高 IoU → 保留大 bbox 那个
filter_candidates() 去掉已识别区域的 rejected quads
Stage 3: Tracking & Fusion
FusionTracker.update() ① IoU 匹配 observations → 已有 tracks
② 匹配成功 → 更新 bbox/corners/missed=0
③ 未匹配 observation → 创建新 track
④ 未匹配 track → missed++
⑤ missed > max_missed → 删除 track
display smoothing: EMA alpha + deadband + snap
Stage 4: Output
可视化 Quad polyline (绿/橙/紫)
Candidate "stag?" 蓝框
轨迹历史线 (颜色=状态)
cv2.imshow / VideoWriter
JSONL 日志 每帧: frame, fps,
observed_ids, skipped_passes,
candidates, tracks

marker_id → angle/position
--tuio --tuio-port 3333

独立窗口 + 视频输出
